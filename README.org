* Enhanced Book Template in Org-mode Using the svmono Class (version: 2.0.0)

The prior version should have been numbered 1.0.0. This version introduces breaking changes with the use of org headlines to ease section folding. It provides a more native org-mode experience.

** What is this for?

Many academic scientists and engineers maintain notebooks of information in LaTeX documents.
They want to capture their insights and new knowledge in a place organized in a hierarchy.
They would like to be able to retrieve that information quickly via the autogenerated table of contents, lists, index, and a PDF of their book document.
They use their notebook in the same way others use personal wikis or zettelkastens to store and retrieve knowledge.

Furthermore, they might have the vain hope of publishing this book someday, but most notebooks are used only for self-study.
Once the skeleton of the book document has been assembled, its style can be easily modified by changing the class to meet the specific needs of a publisher.

*** Template for a dissertation

You could use this Org-Mode book as a template to assemble a dissertation.
Many universities require you to use a specific LaTeX class because they want a uniform style across all dissertations assembled with LaTeX.
You can adapt this template by changing document classes in the preamble of the =main.org= file.
Just change the class name in the Preamble drawer and follow the directions below to add a new class to the available LaTeX classes in the variable ~org-latex-classes~.

** Why Org-mode?

*** Easy table assemby
Org-mode provides excellent support for assembling tables.
There is no peer competitor for the incredible flexibility in assembling tables.
The tables self-adjust by hitting *shift-tab* as you add new columns and rows.
You can do math on columns and rows.
Tables in org-mode are like mini Excel spreadsheets.

*** Interactive programming
Another key feature of Org-Mode is its support for interactive programming inside the org file.
This is an enhanced form of literate programming.
You can turn your code blocks into executable code that generates plots.
This is useful in projects where there is a need to update tables and figures as new data are collected.

*** Thin wrapper around LaTeX
Once you start thinking of Org-mode as a thin wrapper around LaTeX, using it becomes natural.
It is easy enough to retreat to LaTeX if things go wrong.

** What is new in this enhanced version?

This enhanced template reconciles features from the =bookInLaTeX= Overleaf template with the =bookInOrg= template.
The key improvements are:

*** Org-mode idiomatic chapters

All chapter files now use proper org-mode headings (=*=, =**=, =***=) instead of raw LaTeX commands (=\chapter{}=, =\section{}=, =\subsection{}=).
This means that folding, navigation, and org-mode commands work correctly inside chapter files.

*** Comprehensive chapter template with boilerplate

The file =Contents/ChapterTemplate.org= contains a rich set of commented-out boilerplate examples stored in a =:BOILERPLATE:= drawer tagged =:noexport:=.
Copy and paste the examples you need:

- Minted code blocks (with/without line numbers, bash, console, inputminted from file)
- Org-mode source blocks for literate programming
- Equations (numbered, align, split)
- Figures (org-mode syntax and LaTeX syntax)
- Tables (org-mode with booktabs and LaTeX with booktabs)
- Boxy environment (titled highlight box)
- Statistical model float
- Checklist float
- Exercises section
- Epigraphs

*** Expanded preamble

The =main.org= preamble now includes all packages from =preambleOnOverleaf.tex=:

- =natbib= with =cell= bibliography style and =authoryear,round= options
- =authorindex= package with =\listofauthors= command
- =sectsty= with Maroon chapter titles
- =siunitx= for SI units
- =multicol=, =footmisc=, =scrextend=, =array=, =float=
- =url= and =underscore= packages
- Custom float environments: =statsmodel=, =email=, =listeq=, =checklist=, =equ=
- List of Statistical Models infrastructure (=\smcap= and =\listofstatmodels=)
- Boxy environment for titled highlight boxes
- Minted styling with =codebg1=, =codebg2= background colours and =python=/=bash= environments
- Listings package with syntax colouring
- Code colour definitions (=codegreen=, =codegray=, =codepurple=, =backcolour=)
- Symbol list glossary style (=symbunitlong=)
- =unicode-math= (loaded last)
- =filecontents= package
- KOMA-script labelling font

*** Writing log template in org-mode

The file =writingLogTemplate.org= is a full port of =writingLogTemplate.tex=.
It includes sections for rationale, audience, grant aims, related projects, journal lists by impact factor and field, potential titles and keywords, reviewers, competitors, collaborators, draft introduction and abstract, a dated writing log, to-do lists, a word count chart (pgfplots), and a glossary of jargon.

*** Additional support files

- =output-globalEpigraphs.tex= --- Collection of favourite epigraphs to copy into chapter files.
- =main-symbols.tex= --- Symbol list with units (from bookInLaTeX).
- =wordcount.txt= --- Seed data for the word count chart in the writing log.
- =AnnotatedBibliography/= --- Directory for annotated bibliography files (=IEEEannot.bst=, =annot.bib=, =mainAnno.tex=).
- =Contents/codeListings/= --- Directory for external code files imported via =\inputminted=.
- =Contents/figs/= --- Directory for figure image files.

*** Part structure

Parts are now declared with =#+LATEX: \part{The Basics}= in =main.org= rather than via separate =.tex= files, keeping the document structure visible in one place.

** Quick start

1. Clone or copy this directory to start a new book project.
2. Open =main.org= in Emacs.
3. Customize the title, author, date, and bibliography path in the header.
4. Edit the =:PREAMBLE:= drawer only if you need to add or remove LaTeX packages.
5. Edit =Contents/Preface.org= to write your preface.
6. Copy =Contents/ChapterTemplate.org= to create new chapters (e.g., =Contents/chapter3.org=).
7. Add =#+INCLUDE: "./Contents/chapter3.org"= to =main.org= at the appropriate location.
8. Store images in =Contents/figs/= and external code files in =Contents/codeListings/=.
9. Compile with =C-c C-e l o=.  You may need to compile twice to build the bibliography and indices.

** Emacs init.el configuration

The following elisp code will run the required LaTeX programs when you enter ~C-c C-e l o~.
Add it to your Emacs initialization file.
*Lualatex* is recommended when using the minted package for code listings.
*pdflatex* also works with this template if you remove the =luatex= option from hyperref.

#+begin_src elisp
(with-eval-after-load 'ox-latex
  (setq org-latex-pdf-process
        '("lualatex -interaction nonstopmode --shell-escape -synctex=1 -output-directory %o %f"
          "bibtex %b"
          "makeindex -s %b.ist -o %b.ind %b.idx"
          "makeglossaries %b"
          "lualatex -interaction nonstopmode --shell-escape -synctex=1 -output-directory %o %f"
          "lualatex -interaction nonstopmode --shell-escape -synctex=1 -output-directory %o %f")))
#+end_src

*** Adding the svmono class to org-latex-classes

You must add the *svmono* class to the variable *org-latex-classes* after entering *M-x customize*.
Search for =org-latex-classes= and add an entry with the class name =svmono= and its sectioning commands.
See the =images/= folder for a screenshot of the customization menu if available.

** Index entry functions

The book has a subject index, author index, and writing prompt index.
The =imakeidx= package enables three separate indices.
To ease the insertion of index keys, add these functions and keybindings to your =init.el=:

#+begin_src elisp
;;; Index entry insertion functions for imakeidx in org-mode.
(defun ml/insert-main-index-entry ()
  "Insert a general index entry."
  (interactive)
  (let ((term (read-string "Index term: ")))
    (insert "\\index[main]{" term "}")))

(defun ml/insert-author-index-entry ()
  "Insert an author index entry."
  (interactive)
  (let ((author (read-string "Author (Last, First): ")))
    (insert "\\index[authors]{" author "}")))

(defun mooerslab-insert-prompt-index-entry ()
  "Insert a writing prompt index entry."
  (interactive)
  (let ((prompt (read-string "Writing prompt: ")))
    (insert "\\index[prompt]{" prompt "}")))

(eval-after-load 'org-mode
  '(progn
     (define-key org-mode-map (kbd "C-c w a") 'ml/insert-author-index-entry)
     (define-key org-mode-map (kbd "C-c w m") 'ml/insert-main-index-entry)
     (define-key org-mode-map (kbd "C-c w p") 'mooerslab-insert-prompt-index-entry)))
#+end_src

** Org-mode heading conventions for chapters

Chapters use org-mode headings that map to LaTeX sectioning as follows:

#+CAPTION: Mapping of org-mode headings to LaTeX sectioning commands.
#+ATTR_LATEX: :booktabs t
| Org heading level | LaTeX equivalent  | Usage                   |
|-------------------+-------------------+-------------------------|
| =*=               | =\chapter{}=      | Chapter title           |
| =**=              | =\section{}=      | Major section           |
| =***=             | =\subsection{}=   | Subsection              |
| =****=            | =\subsubsection{}= | Sub-subsection (rare)   |

LaTeX-specific commands such as =\epigraph=, =\index=, and =\label= are placed on lines immediately following the org heading.
They are passed through to LaTeX verbatim during export.

** Tips

- Use =:noexport:= tag on headings to exclude them from the PDF (e.g., the boilerplate drawer in ChapterTemplate.org).
- Use =:GUIDANCE:= drawers for in-document help that stays hidden during normal editing.
- Test the compilation pipeline early --- after writing two or three chapters --- to catch formatting issues.
- Keep the number of active packages manageable; comment out packages you do not need to reduce compile time and potential conflicts.
- Start a writing log (=writingLogTemplate.org=) at the same time you start the book project.

** Version control

Because Org projects are plain-text folders, they work seamlessly with Git.
Initialize a repository at the project root:

#+begin_src shell
cd /path/to/your-book-project
git init
git add -A && git commit -m "Initial book project structure"
#+end_src

Consider using the Emacs package =magit= for automatic commits.
